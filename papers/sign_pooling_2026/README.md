# Gated Cluster-Pooled Sign Constraints for Multi-Output Sparse Regression

Replication repository for Sepp & Kastenholz, *Gated Cluster-Pooled Sign
Constraints for Multi-Output Sparse Regression* (submitted to *Computational
Statistics & Data Analysis*). The method is implemented in the
[`factorlasso`](https://pypi.org/project/factorlasso/) package; this repository
holds the manuscript source and the code that regenerates every figure and
table.

## Layout

```
sign_pooling_2026/
├── Makefile
├── paper/                       LaTeX submission bundle (compiles as-is)
│   ├── article.tex  refs.bib
│   ├── cas-sc.cls  cas-common.sty  cas-model2-names.bst  thumbnails/
│   ├── figS1..S5_*.pdf  figE1..E3_*.pdf      (committed; regenerable)
│   └── tableS1,S2_*.tex  tableE1_*.tex
└── replication/
    ├── requirements.txt
    ├── sign_pooling_simulation.py   exhibits.py        (simulation -> Fig/Table S*)
    ├── sign_pooling_robustness.py                      (robustness -> Fig S5)
    ├── eqtl_pipeline.py             eqtl_exhibits.py   (eQTL -> Fig/Table E*)
    ├── data/         yeast_full.npz  yeast.rda  (public, ~3.5 MB)
    └── results/      cached CSVs (sim + eQTL), regenerated by the scripts
```

## Build the paper

```
make paper          # pdflatex + bibtex + pdflatex x2, in paper/
```

The figures and tables are committed, so the paper builds without running any
code. It uses the Elsevier CAS class (vendored). The manuscript is in review
(double-spaced) mode; change `\documentclass[...]{cas-sc}` to the `[final]`
option for the compact camera-ready layout.

## Reproduce the results

```
pip install -r replication/requirements.txt
make sims           # simulation at R = 200; regenerates Fig S1-S5, Tables S1-S2 (~120 s)
make eqtl           # eQTL pipeline on the bundled data; regenerates Fig E1-E3, Table E1
make figures        # both
make all            # figures, then the paper
```

The simulation is deterministic (each replication is seeded by its index). The
sign metrics are computed from the closed-form gate of `factorlasso` directly,
with no cone solve, so the full R = 200 sweep runs in about a minute. A faster
pass: `python replication/sign_pooling_simulation.py 50`.

## Script → exhibit map

| Script                       | Outputs |
|------------------------------|---------|
| `sign_pooling_simulation.py` | `results/{base_table, recoverability, gate_roc, regime_map, sign_consistency}.csv` |
| `sign_pooling_robustness.py` | `results/{correlated_predictors, rhocut_sensitivity, gate_correction}.csv` |
| `exhibits.py`                | Fig S1–S5, Tables S1–S2 (written into `paper/`) |
| `eqtl_pipeline.py`           | `results/{clusters, derived_signs, hotspots, parity, summary}.csv`, `fit.npz` |
| `eqtl_exhibits.py`           | Fig E1–E3, Table E1 (written into `paper/`) |

## Data

Yeast eQTL cross of Brem & Kruglyak (2005), public, redistributed with the
`trigger` Bioconductor package. See `replication/data/README.md`.
